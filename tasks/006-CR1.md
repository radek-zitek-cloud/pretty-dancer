## Reason

`AIMessage.usage_metadata` from OpenRouter contains `input_tokens`,
`output_tokens`, and `total_tokens`. `input_tokens` represents the full context
size — system prompt + entire message history + current input — which is the key
metric for understanding how conversation history grows across turns. Capturing
this alongside the existing `llm_trace` emit costs three lines and makes
`show_run.py` significantly more useful.

---

## Changes

### 1. `src/multiagent/core/agent.py` — capture usage after LLM response

In `call_llm`, after `output = str(response.content)` and before the
`llm_trace` block, add:

```python
usage = response.usage_metadata or {}
self._log.debug(
    "llm_usage",
    input_tokens=usage.get("input_tokens"),
    output_tokens=usage.get("output_tokens"),
    total_tokens=usage.get("total_tokens"),
    history_length=len(state["messages"]),
)
```

`llm_usage` is always emitted at `DEBUG` — it appears in the JSONL file when
`log_json_file_level=DEBUG` (the default) and is suppressed from console at
`INFO` level. It does not require `log_trace_llm=True`.

### 2. `src/multiagent/core/agent.py` — add token fields to `llm_trace`

Extend the existing `llm_trace` emit to include token counts:

```python
if self._settings.log_trace_llm:
    self._log.info(
        "llm_trace",
        prompt=state["messages"][-1].content,
        system_prompt=self._system_prompt,
        response=output,
        history_length=len(state["messages"]),
        input_tokens=usage.get("input_tokens"),
        output_tokens=usage.get("output_tokens"),
        output_chars=len(output),
    )
```

### 3. `scripts/show_run.py` — surface token counts in LLM calls section

In the LLM calls section (Section 3), add `input_tokens` and `output_tokens`
to each call panel header alongside the existing char counts.

Add a summary row at the end of Section 3:

```
Total: {n} LLM calls  |  {sum(input_tokens)} input tokens  |  {sum(output_tokens)} output tokens
```

If `usage_metadata` was unavailable for any call (fields are `None`), omit
the totals row and print:

```
Token counts unavailable for one or more calls.
```

---

## Availability note

`usage_metadata` is populated by OpenRouter for most models. If the field is
`None` on the response object, the `or {}` guard returns an empty dict and all
`.get()` calls return `None`. Logging `None` values is intentional — it makes
the absence visible in the JSONL rather than silently omitting the fields.
