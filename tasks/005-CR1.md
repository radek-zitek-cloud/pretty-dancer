---

## Reason

When multiple agents run concurrently, per-run log files named by timestamp alone
are ambiguous. The progressive and conservative agents start within the same second
and produce colliding filenames. Agent name must be included in every log filename.

---

## Changes

### 1. `configure_logging()` — add `agent_name` parameter

Add `agent_name: str = ""` as the second positional parameter, between `settings`
and `experiment`:

```python
# BEFORE
def configure_logging(
    settings: Settings,
    experiment: str = "",
) -> tuple[Path | None, Path | None]:

# AFTER
def configure_logging(
    settings: Settings,
    agent_name: str = "",
    experiment: str = "",
) -> tuple[Path | None, Path | None]:
```

Update the docstring Args section:

```
agent_name: Name of the running agent. Always included in log filenames
    so concurrent agents produce separate, identifiable files.
```

### 2. Filename construction — include agent name always

Change the filename pattern from:

```
{timestamp}[_{experiment}].{ext}
```

to:

```
{timestamp}_{agent_name}[_{experiment}].{ext}
```

Agent name is always present. Experiment label is still optional (omitted when empty).

Concrete examples:

```
logs/2026-03-13T14-32-01_progressive.log
logs/2026-03-13T14-32-01_progressive.jsonl
logs/2026-03-13T14-32-01_conservative.log
logs/2026-03-13T14-32-01_conservative.jsonl
logs/2026-03-13T15-01-44_progressive_debate.log
logs/2026-03-13T15-01-44_progressive_debate.jsonl
```

### 3. `src/multiagent/cli/run.py` — pass `agent_name`

```python
# BEFORE
human_log, json_log = configure_logging(settings, experiment=experiment)

# AFTER
human_log, json_log = configure_logging(settings, agent_name=agent_name, experiment=experiment)
```

`agent_name` is already available as the CLI argument — no new logic required.
