## Reason

`show_thread.py` requires knowing a thread_id before you can use it. In practice,
the developer looks up thread_ids from the database manually before running the
script. `browse_threads.py` eliminates that step — it displays all threads as a
numbered list and launches `show_thread.py` on the selected one.

## Changes

### 1. Create `scripts/browse_threads.py`

New fourth inspection script. Uses `sqlite3`, `subprocess`, `rich`, and
`Settings().sqlite_db_path` — no new dependencies.

**Behaviour:**

1. Load database path from `Settings().sqlite_db_path`. If settings load fails,
   print a clear error and exit 1.
2. Query thread summary — see query below.
3. Display as a numbered `rich.table.Table`.
4. Prompt: `Enter thread number (or q to quit): `
5. On valid selection: call `show_thread.py` with the full thread_id via
   `subprocess.run(["uv", "run", "python", "scripts/show_thread.py", thread_id])`.
6. After `show_thread.py` exits, redisplay the table and prompt again.
7. On `q` or empty input: exit 0.
8. On invalid input: print `Invalid selection.` and re-prompt without redisplaying
   the table.

**Thread summary query:**

```sql
SELECT
    thread_id,
    COUNT(*)                                       AS message_count,
    SUM(processed_at IS NOT NULL)                  AS processed_count,
    MIN(created_at)                                AS started_at,
    MAX(created_at)                                AS last_at,
    MIN(body)                                      AS preview
FROM messages
GROUP BY thread_id
ORDER BY MAX(created_at) DESC
```

**Table columns:**

| # | Thread ID | Messages | Processed | Preview | Started | Last activity |
|---|---|---|---|---|---|---|

- `#` — selection number, 1-based
- `Thread ID` — first 8 chars of UUID only (full value used internally)
- `Messages` — total message count in thread
- `Processed` — count where `processed_at IS NOT NULL`
- `Preview` — first 60 chars of `MIN(body)`, ellipsis if truncated
- `Started` — `MIN(created_at)`, formatted as `HH:MM:SS` (date omitted for brevity)
- `Last activity` — `MAX(created_at)`, same format

If no threads exist, print `No threads found in database.` and exit 0.

**Module docstring:**

```python
"""Browse conversation threads from the SQLite transport database.

Displays all threads as a numbered list and launches show_thread.py
on the selected thread. Database path is read from application settings.

Usage:
    uv run python scripts/browse_threads.py
    just threads
"""
```
